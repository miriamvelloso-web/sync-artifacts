When an initiative spans multiple markets (e.g., KW, UAE, EG, QA), break down the update BY MARKET with separate status per market. Do NOT lump all markets into a single bullet.

Also: NEVER mix initiatives. Flash Sale (TLBOPS-13/Howaida) is completely separate from F&UF buying signals (TLBOPS-7/Firas), even if both involve grocery in the same market. Use the PM mapping to correctly attribute Slack context to the right initiative.

**Why:** Miriam caught Flash Sale KW data (+36.6% lift) posted under TLBOPS-7 (F&UF). That data belongs to TLBOPS-13. Each initiative has its own PM — use that to filter correctly.

**How to apply:**
1. When searching Slack, filter by PM name + initiative keywords to avoid cross-contamination
2. Structure each multi-market initiative with market headers (EG / UAE / KW / QA)
3. If a market has no activity for that initiative, say "No significant activity this week"
4. Double-check every data point: does it belong to THIS initiative's PM, or another?
