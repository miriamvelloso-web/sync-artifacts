Claude Code Skill System

One input, all systems updated.

A skill system for Product Managers at Talabat. Sync initiative updates across Jira, Google Docs, Google Slides, Slack, and email — from a single input.

/sync [paste your update here]
Commands
Everything starts with /sync

On first run, onboarding discovers the PM's Jira project, maps initiatives to epics, and registers artifact destinations. After that, the onboarded PM is always used — no selection prompts.

/sync [updates]
Parse updates from any input (text, images, PDFs, Slack messages), route to all registered destinations — Jira, Docs, Slides, Slack, email.
/sync artifacts
Generate or update branded Google Slides decks. Shows a dynamic menu with Experiment, Lightspeed, MPR, OKR Slides, registered decks, and + New deck.
/sync weekly
Draft weekly CEO update in Toon's 3-section format — what shipped, focus next week, blockers. Pulls context from Jira, Slack, Drive, Gmail.
/sync add [URL | slack | email]
Register a new destination — Google Docs, Slides URLs, Slack channels, or email routes.
/sync status
Show the current sync registry — all destinations, mapped initiatives, routing rules.
/sync remove [name]
Remove a destination from the registry.
/sync test [dest]
Send a test message to verify connectivity with a specific channel.
Architecture
Parse → Route → Update → Verify

The parse layer, artifact layer, and channel layer are independent. Adding a new destination means registering it — nothing else changes.

    How it works
    /sync [updates] — The Update Flow

    From any input format to all destinations in one pass.

    Parse

    Extract structured updates per initiative from text, images, PDFs, Slack messages, meeting notes — any format.

    Load Registry

    Identify affected initiatives, Jira epics, mapped artifacts, and notification channels.

    Route

    Build a routing plan showing every destination that will be updated. Present for PM approval.

    Update Jira

    Comments on epics, status transitions, new tickets from action items, field edits on descriptions.

    Update Artifacts

    Google Docs: inline field edits + status log append. Google Slides: update existing or create new slides.

    Notify Channels

    Slack posts, thread updates, DMs. Email notifications, digest summaries. Links to already-updated artifacts.

    Verify & Report

    Re-read modified sections, confirm Jira comments posted, report failures. Summary table with checkmarks.

    First run
    Onboarding

    Automatic setup on first /sync invocation. Discovers Jira structure, registers artifacts, maps initiatives.

    1. Identify Jira Project

    Confirm or ask for the project key. Fetch full epic/initiative hierarchy.

    2. Register Artifacts

    Accept Google Doc/Slides URLs. Verify access. Discover tabs, slides, sections.

    3. Map Initiatives

    Link sections/tabs/slides to Jira epics. Natural language: "SR tab → HOMESQUAD-967".

    4. Register Channels

    Optional. Slack channels and/or email with trigger levels and format preferences.

    5. Context Sources

    Slack channels, Drive docs, Gmail contacts per initiative for weekly gathering.

    6. Save Registry

    Write per-PM YAML config to Claude memory. Confirm setup complete.

    /sync weekly
    Weekly CEO Update (Toon Format)

    Scans the PM's Jira board, gathers context from Slack, Drive, and Gmail, and drafts a CEO-scannable HTML email.

    1. Scan

    Fetch active initiatives from Jira. Identify what changed in the last 7 days.

    2. Gather Context

    Pull from Slack channels, Google Drive docs, Gmail threads per initiative. All in parallel.

    3. Draft

    Squad + PM header with OKR/KR progress table. Three sections: shipped, next week focus, blockers & support needed.

    4. Review

    Present draft for PM approval before sending. Iterate if needed.

    Data Model

    Channel Trigger Levels

    • all — every /sync invocation
    • initiatives — specific initiatives only
    • significant — launches, blockers, scope changes
    • milestone — major milestones only

    Channel Format Levels

    • detailed — full context, metrics, links, action items
    • summary — brief summary with links
    • executive — one-liner with key metric
    • digest — multi-initiative roundup

    Guardrails
    Rules

    Operational rules that keep the system consistent and predictable.

    1

    Always use /tlb-slides — never create slides without the Talabat brand design system.

    2

    Always use the onboarded PM — never ask which PM; use the one from the sync registry.

    3

    MPR uses the master template — duplicate via copy_drive_file, never recreate from scratch.

    4

    Inline first, log second — edit canonical fields before appending status sections.

    5

    Index discipline — process edits from highest index to lowest within each tab.

    6

    Preserve formatting — re-apply text style after insertion in formatted cells.

    7

    All destinations or explicit failure — every registered destination must be updated or reported as failed.

    8

    Create new tickets from action items — if an update mentions an action item with an owner, create a Jira task.

    9

    Artifacts = Google Slides only — /sync artifacts is exclusively for slide decks.

    10

    Create or update, never duplicate — check if a deck exists before creating; update in place if it does.

    11

    Organize in Drive folders — all generated decks go into Q{q}Y{yy} PM: {Name} BET: {Bet}/{Type}/

    12

    Registry is truth — all mappings live in the sync registry memory file.
